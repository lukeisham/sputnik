### **Phase 2: Fix Feature Gaps**

#### **ISS-042** — Markdown Preview: Full Syntax Rendering
- **File:** `4 Markdown Preview/MarkdownPreviewViewModel.swift:88`
- **Problem:** Parser uses `.inlineOnlyPreservingWhitespace`, so headings, tables, lists, blockquotes render as literal source text
- **Guide Requirement:** "headings … tables, lists, and block quotes rendered"
- **Solution:**
  1. Switch to `.full` parsing in `MarkdownPreviewViewModel.render(markdown:baseDir:)`
  2. Add a custom intent→attribute styling pass to map `PresentationIntent` (which `AttributedString(markdown:)` emits) to visual attributes `NSTextView` recognizes:
     - `heading(level)` → bold + larger font
     - `codeBlock` → monospace background
     - `blockQuote` → left border + gray
     - `unorderedList`, `orderedList` → bullet/number prefix
     - `table` → `NSTableView` or multi-line cell formatting (complex; may defer to grid-like layout)
  3. Test with a `.md` file containing each element type
- **Effort:** ~2–3 hours (presentation intent styling is new code)
- **Owner:** Module 4 lead
- **Test:** Open a markdown file with headings, lists, blockquotes, tables; verify they render, not source

---

#### **ISS-043** — HTML Preview: Selection Capture at Runtime
- **File:** `8 HTML Preview/HTMLPreviewView.swift:131`, `HTMLPreviewCoordinator.swift`
- **Problem:** `allowsContentJavaScript = false` may suppress injected user scripts, so `capturedSelection` likely never populates
- **ISS-010 Status:** Marked resolved but needs runtime verification
- **Solution:**
  1. Build the project
  2. Open an HTML file in the preview panel
  3. Select text in the HTML preview
  4. Right-click → "More Context"
  5. Verify that the selected word is captured and the help lookup fires
  6. If it fails:
     - Check whether `configuration.defaultWebpagePreferences.allowsContentJavaScript = false` applies globally or per-script
     - If global: may need to conditionally allow JS only for the selection script + block all other JS with CSP
     - If per-script: verify that `addUserScript()` runs before JS is locked down (likely order issue)
- **Effort:** ~1 hour (verify only; if broken, debugging may take 2–3 more hours)
- **Owner:** Module 8 lead
- **Verification:** Must be able to capture text selection via right-click context menu
